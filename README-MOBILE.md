# Ledger on mobile — Android & iOS (Capacitor)

This project is wrapped with **[Capacitor](https://capacitorjs.com/)**, which runs your existing web build inside a native app shell. **No code rewrite** — the same React/Vite app you run on the web becomes the Android and iOS app. Two things are upgraded automatically when running natively:

- **Reminders become real scheduled notifications** (delivered by the OS even when the app is closed), via `@capacitor/local-notifications`.
- **Data durability**: every save is mirrored to native storage (`@capacitor/preferences`) and restored on launch, so your finances survive even if the WebView clears its localStorage.

Both are guarded by a platform check, so the plain web build is unchanged.

---

## Prerequisites

- **Node.js 18+** (for everything).
- **Android:** [Android Studio](https://developer.android.com/studio) (includes the Android SDK) and **JDK 17**.
- **iOS:** a **Mac** with **Xcode 15+** and **CocoaPods** (`sudo gem install cocoapods`). iOS apps cannot be built on Windows/Linux — that's an Apple requirement, not this project's.

> I prepared and validated the web bundle and all the Capacitor wiring, but native builds need Android Studio / Xcode and platform SDKs, so they run on your machine — not in this chat.

## One-time setup

```bash
# 1. install web + Capacitor dependencies
npm install

# 2. build the web app (outputs to /dist, which Capacitor uses)
npm run build

# 3. add the native platforms (creates the android/ and ios/ folders)
npx cap add android
npx cap add ios          # Mac only

# 4. generate app icons + splash screens from /assets
npm run cap:assets
```

`appId` and `appName` live in `capacitor.config.ts` (currently `com.ledgerapp.finance` / `Ledger`). Change the `appId` to your own reverse-domain identifier **before** publishing.

---

## Run it

Each command rebuilds the web app, copies it into the native project, and opens the IDE:

```bash
npm run android      # build + sync + open Android Studio
npm run ios          # build + sync + open Xcode (Mac only)
```

Then press **Run** in Android Studio / Xcode to launch on an emulator/simulator or a connected device.

To run straight onto a device from the terminal:

```bash
npm run run:android
npm run run:ios
```

### After you change web code

```bash
npm run cap:sync      # = npm run build && cap sync
```

…then Run again in the IDE. (`cap sync` also reinstalls native plugins.)

## Notifications on device

- The app requests permission when you toggle a reminder on in **Settings → Reminders**.
- **Android 13+** requires the `POST_NOTIFICATIONS` runtime permission — the local-notifications plugin requests it for you; no manual manifest editing needed.
- **iOS** shows the system permission prompt on first request.
- Reminders are scheduled as **daily repeating** local notifications at the times you choose, and fire even when the app is closed.

## Release builds & store submission

### Android (Google Play)
1. In Android Studio: **Build → Generate Signed Bundle / APK → Android App Bundle (.aab)**. Create/keep a keystore (back it up — losing it means you can't update the app).
2. Set the version in `android/app/build.gradle` (`versionCode`, `versionName`) for each release.
3. Upload the `.aab` to the [Play Console](https://play.google.com/console) (one-time \$25 developer fee). Fill in the store listing, screenshots, and the Data Safety form.

### iOS (App Store)
1. In Xcode: set your **Team** (Signing & Capabilities) and a unique **Bundle Identifier** matching `appId`.
2. Set version/build under the target's **General** tab.
3. **Product → Archive**, then distribute via the Organizer to **App Store Connect** ([Apple Developer Program](https://developer.apple.com/programs/) is \$99/yr). Add screenshots and complete the App Privacy questionnaire.

### Privacy (easy here)
Ledger stores everything **on the device** and sends nothing to any server. For both stores' privacy forms you can declare **no data collected**. You'll still need a hosted privacy policy URL — a short page stating "all data stays on your device; nothing is collected or transmitted" is sufficient.

## Optional: live reload during development

For instant refresh on a device while coding, point the native app at your dev server. In `capacitor.config.ts` temporarily add:

```ts
server: { url: 'http://192.168.1.X:5173', cleartext: true }
```

Run `npm run dev -- --host`, then `npx cap run android`/`ios`. **Remove this before building a release** so the app loads its bundled files.

## Notes

- The `android/` and `ios/` folders are generated by `npx cap add` on your machine and aren't included in the zip. They're yours to commit to source control once created.
- The PWA service worker is redundant inside the native app (assets are already local) and stays inert there; it continues to power offline use on the web build.
- To change the app icon/splash later, replace the images in `/assets` and re-run `npm run cap:assets`.
